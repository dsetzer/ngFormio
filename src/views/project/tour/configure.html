• 1
    Configure the app

    This step helps configure your application so that it is easy to manage the forms and stages for your project.

    The StarterKit app and our general recommendation is to have a config.js file typically located at /src/config.js. You can see an example at https://github.com/formio/ng-app-starterkit/blob/master/src/config.js.

    You can locate the Project URL of your project in the upper right hand corner where it says Live Project Url. This Project URL is shared by all the forms and resources within your project.

    Projects that are on the Enterprise plan can have multiple stages. Each stage has a different Project URL. By creating a config file and referencing every form with this Project URL variable it is easy to change which Stage the application is pointed at.

    To set your project in the application you have set up, change the first line of /src/config.js to say the following.

    var APP_URL = '{{ variables.projectUrl }}';

    Notice that all the form urls should now be pointing to that project's forms. If you change the APP_URL to a different stage, all the forms will change as well.

  • 2
    Running the app

    You can run a development version of the application from the command line.

    gulp serve

    This should open up a browser window and refresh the page each time you make changes to the application.

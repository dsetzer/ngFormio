About Projects

A project serves as a container for the Resources, Forms, API, and Data for a specific application you are building. A project can serve as the backend API platform for your application. Here is what you need to know to get started.

Resources

Resources are the main data structures that can serve as the backend storage and API platform for your web application. Resources can also be embedded and linked into other Resources and Forms to create a hierarchical relationship structure. Each Resource has an associated Resource Form.

Forms

Forms serve as the main interfaces for your application. You can re-use Resource fields as well as use custom input components. The forms you create can range from simple surveys to complex authentication and resource management forms. Forms are what are embedded within your application.

API

Each form has an API automatically built behind it. Each form will automatically connect to its corresponding API. However, the API is fully available should you want to directly interact with it. The API explorer allows seeing which endpoints are available and how to interact with them.

Data

Data is where you can find and manage all the data within your application. Here you can view charts of analytics data on how many submissions your users are making over time. You can also access an interactive table of all submissions for your forms that you can sort, filter, and more.

Preview

Preview your working application live. Any changes made in your Forms and Resources will immediately be reflected in the preview application. This serves as an immediate preview of your application before you download and begin custom development and launch your application.

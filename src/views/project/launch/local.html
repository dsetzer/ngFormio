Set Up Local Development

In order to do local development you will need to have node.js and npm installed. See nodejs installation instructions.

Step 1 - Get the source code

To begin development of this application you will need to set up a local development environment. Start by either downloading or cloning the github repo.

Download the repo

Downloading the repo does not require git but also does not track and manage changes you make unless you set it up manually with a new git repo.

Go to {{ repo }}

Click the Download Zip button.

Unzip the file, open your terminal and change directory to the app directory.

Clone the repo

In order to clone the github repo you must have git installed on your computer. If you don't already have it, you can get it at https://git-scm.com/.

Once you have git, go to your terminal and type

git clone {{ repo }}.git
cd {{ appFolder }}

Step 2 - Install dependencies

This step requires that that nodejs, npm and bower are installed. Bower can be installed after nodejs and npm by typing this in the terminal

npm install -g bower

Install both node and bower dependencies by typing this in your terminal from the application base directory.

npm install
bower install

Step 3 - Configure for your project

In order for the app to use your form.io project, you need to configure the app to know where it is.

Change the first line of /src/config.js to say the following.

var APP_URL = '{{ projectUrl }}';

Step 4 - Running the app

You can run a development version of the application from the command line. This will start up a server and a browser window and refresh the page any time you change the source code.

gulp serve

Then add any form using the following syntax.

<formio src="'{{ projectPath({name: 'myproject'}) }}/yourform'"></formio>

Embed Forms

Forms are embedded within your application by using the <formio> directive and referencing the form URL endpoint.

Using these embed codes you can perform full CRUD and index on your form submissions.

More information on embedding can be found at Angular Embedding Help

Basic embedding

This would embed one of your forms and allow the data to be submitted back to the API server.

<formio src="{{ projectUrl }}/{{ forms[0].path }}"></formio>

List of Submissions

It is often useful to list the submissions already sent to a form backend. This can be done simply using the <formio-submissions> directive.

<formio-submissions src="{{ projectUrl }}{{ forms[0].path }}"></formio-submissions>

Form Update

You can also reference an existing submission in the form url and the data from that submission will be loaded along with the form. When the form is submitted, an update will happen instead of a create.

<formio src="{{ projectUrl }}/{{ forms[0].path }}submission/{submissionId}"></formio>

Form Submission Delete

Finally, you can allow users to delete a submission with the <formio-delete> directive. If the user has permission, they can delete the submission.

<formio-delete src="{{ projectUrl }}/{{ forms[0].path }}submission/{submissionId}"></formio-delete>

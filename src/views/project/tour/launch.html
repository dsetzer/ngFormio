There are two ways you typically want to start your application. First you'll start it in development mode while you work on building it out. Then once it is ready, build it and launch it to a server.

  • 1
    Local Development Environment

    If you are using one of our applications as a base, we provide a gulp task that will server a development version of the application.

    gulp serve

    This will set up the app, start the server and open a browser window for you. Any changes you make within the applications source code will trigger a refresh. This makes development faster and easier.

    Once launched, you can log in as the Admin User you created earlier if you application has Authentication configured for it.

  • 2
    Build a Distribution

    To build your app and prepare it for hosting, you will need to compile it. If you are using one of our applications as the base, we provide a gulp task that will do this for you.

    gulp build

    This will build your entire application into the /dist directory. This will be a fully working version of your app that is ready for deployment.

    Deploy to Hosting

    Form.io based apps can be hosted anywhere that is able to serve files over http/s. Since all of the logic and functionality is done on the form.io backend, there are no special requirements for hosting the app. There are many good free hosts you can use to host your app or set up your own server.

    To deploy your app, make sure you have run gulp build and copy the contents of the /dist directory to your server host.

    We host many of our apps using github pages which is free.

    Amazon s3 with cloudfront is a very good high performance alternative.

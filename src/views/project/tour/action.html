• 1
    Go to form actions page
    To add an action, go to the Demographics Form and click on the Actions tab at the top.
    You should already see a Save Submission action. All new forms automatically get this action which instructs the API to save the submission to the database. If you remove this action it will no longer save anything when the form is submitted. Leave it there for this walkthrough.
  • 2
    Create an action

    Select Email from the Select an Action dropdown

    Click the Add Action button to start adding a new action.

  • 3
    Format the email action

    In the To: Email Address field, add in the email field: {{'{{'}}data.email}}. This must match the API Property of the Email field we added to our form.

    The data variable is available with all of the data that was submitted from the form fields and you can use {{'{{'}}}} to indicate replacements. In this case we are going to send the email to whatever email address was entered in the form.

    You can also modify the From, Subject and the Message fields or leave them as defaults.. We provide a submission() macro automatically that will print out a table of the data values.

  • 4
    Save and test the email action

    Click the Save Action button to finish adding the action to your Demographics form. Now every time a new submission occurs to the form, the data is emailed to the email address on the form.

    Go to the Use tab to test it out.

    After submitting the form, you should receive an email (provided you entered your email address) and you should see the submission in the Data tab.

You now know your Demographics form is working and an email will be sent on submission. Now lets look at how to add this form to your application.
